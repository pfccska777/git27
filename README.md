# Решение 1

Режимы репликации master-slave и master-master, их различия

Master-Slave репликация

- Описание

В классической архитектуре один сервер выступает мастером (master), который принимает все записи и изменения данных, а один или несколько серверов слейвов (slave) получают эти изменения и синхронизируют свои данные.

- Особенности
- Мастер — единственный источник записи.
- Слейвы только читают данные и непрерывно получают обновления с мастера.
- Используется для распределения нагрузки на чтение и резервного копирования.
- Репликация односторонняя (мастер → слейв).

- Преимущества
- Простота настройки и управления.
- Хорошо подходит для масштабирования чтения.

- Недостатки
- Нет возможности записи на слейве.
- Точка отказа — мастер.

Master-Master репликация

- Описание

Два или более сервера одновременно являются мастерами и слейвами друг для друга. Каждый сервер может принимать изменения, которые затем реплицируются на остальные узлы.

- Особенности
- Каждый мастер может принимать записи.
- Обеспечивает двунаправленную (или многонаправленную) репликацию.
- Требует тщательной организации для предотвращения конфликтов данных (например, уникальные server-id, синхронизация auto-increment и т.п.).

- Преимущества
- Высокая доступность и отказоустойчивость.
- Можно писать на любой из серверов.
- Подходит для геораспределённых систем.

- Недостатки
- Сложнее в настройке и отладке.
- Риск конфликтов при одновременной записи.
- Требует дополнительные механизмы для разрешения конфликтов.
