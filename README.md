### Решение 2

- Отличия режимов сканирования SYN, FIN, Xmas, UDP с точки зрения сетевого трафика и ответа сервера

SYN-сканирование
Трафик: клиент отправляет TCP-пакет с флагом SYN (установка соединения).

- Ответ сервера:

Если порт открыт — сервер отвечает SYN+ACK, что означает готовность установить соединение.

Если порт закрыт — сервер отвечает RST (сброс соединения).

- Особенность: не завершает полное TCP-соединение (удерживает "полуоткрытое" соединение), что уменьшает вероятность обнаружения (полуоткрытый скан).

FIN-сканирование
Трафик: клиент отправляет TCP-пакет с установленным флагом FIN (завершение соединения).

- Ответ сервера:

Если порт закрыт — сервер отвечает RST.

Если порт открыт — сервер игнорирует пакет (нет ответа).

- Причина: согласно TCP-стандарту, FIN на закрытом порту вызывает сброс, на открытом — отсутствие ответа.

- Особенность: некоторые брандмауэры и системы фильтрации неправильно обрабатывают эти пакеты, что помогает скрытно сканировать.

Xmas-сканирование
Трафик: клиент отправляет TCP-пакет с флагами FIN, PSH и URG (пакет "ёлочная гирлянда").

- Ответ сервера:

Если порт закрыт — сервер отвечает RST.

Если порт открыт — нет ответа.

- Причина: аналогично FIN-скану, но с дополнительными флагами.

- Особенность: используется для обхода некоторых систем фильтрации, менее распространён.

UDP-сканирование
Трафик: отправляется UDP-пакет на выбранный порт (без установки соединения, без подтверждения).

- Ответ сервера:

Если порт закрыт — сервер отвечает ICMP "port unreachable" (порт недоступен).

Если порт открыт — обычно отсутствует ответ (UDP без подтверждений).

- Особенность: сканирование медленное и менее надежное из-за отсутствия подтверждений, часто сложно отличить потерю пакетов от отсутствия ответа.

- Итог для Wireshark

SYN-скан сгенерирует входящий SYN+ACK или RST — чёткие ответы.
FIN/Xmas — либо RST в ответ, либо отсутствие ответа от открытых портов.
UDP — либо ICMP-сообщения об ошибках, либо тишина.
Это создаёт разные шаблоны трафика, что влияет на методику обнаружен

# Пример сканирований для Metasploitable на Debian 12

1. **Скан SYN (полусканирование)**  
```bash
sudo nmap -sS 1.1.1.1

Проверяет открытые TCP-порты быстро и скрытно.

2. Финальный скан (FIN scan)

sudo nmap -sF 1.1.1.1

Отправляет FIN-пакеты, чтобы обнаружить открытые/закрытые порты.

3. Xmas-скан (Xmas scan)

sudo nmap -sX 1.1.1.1

Отправляет пакеты с флагами FIN, PSH, URG, чтобы выявить статус портов.

4. UDP-скан для порта 53 (DNS)

sudo nmap -sU -p 53 1.1.1.1

Проверяет открытость UDP-порта 53.

Нужно заменить 1.1.1.1 на IP-адрес вашей Metasploitable
